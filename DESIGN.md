# DESIGN.md — 타이어 상품 상세 카드 디자인 시스템

> **이 파일은 디자인 소스 오브 트루스(Source of Truth)입니다.**  
> 컴포넌트를 추가·수정할 때 이 파일의 원칙을 먼저 확인하세요.

---

## 1. 제품 컨셉 & 디자인 방향

### 한 마디로 표현하면
> **"한 장의 포스터로 보는 타이어 인포그래픽 카드"**

### 목표 사용자 경험
- 어드민이 등록한 데이터(차종·계절·등급·사이즈·성능 수치)가 **즉시 눈에 들어온다**
- 한국 타이어 이커머스(tirepick, blackcircles.co.kr) 수준의 **신뢰감 있는 정보 밀도**
- html2canvas로 추출하는 **PNG 카드**가 인쇄물처럼 깔끔하게 나온다
- 편집 모드: 클릭하면 바로 인라인 수정 가능

### 레이아웃 철학
| 원칙 | 설명 |
|------|------|
| **인포그래픽 우선** | 숫자·아이콘·차트로 먼저, 긴 설명문은 최소화 |
| **섹션 구분선** | 헤더/푸터 강조 바(Bar) + 내부 구분선으로 리듬감 |
| **타이어 이미지 중심** | 타이어 이미지가 포인트 레이아웃의 앵커 역할 |
| **흑백 베이스 + 포인트 컬러** | 흰 배경 + 검정 타이포 + 빨간 강조가 기본 팔레트 |

## 3. 타이포그래피

### 폰트 패밀리
```
font-family: "Pretendard", "Noto Sans KR", system-ui, -apple-system, sans-serif;
```

> Pretendard는 `public/fonts/`에 woff 9종 탑재, `globals.css`에 @font-face 등록됨.

### 타입 스케일

| 역할 | 크기 | 굵기 | 용도 |
|------|------|------|------|
| Display | 56–72px | 900 (Black) | 히어로 모델명 |
| H1 | 32–40px | 800 (ExtraBold) | 섹션 포스터 헤드카피 |
| H2 | 22–26px | 700 (Bold) | 섹션 제목, 스펙 수치 |
| H3 | 16–18px | 600 (SemiBold) | 카드 레이블, 서브타이틀 |
| Body | 14–15px | 400 (Regular) | 설명문, 리뷰 |
| Caption | 11–12px | 400–500 | 배지, 단위, 보조 레이블 |

### 핵심 규칙
- **숫자는 항상 Black(900) 또는 ExtraBold(800)** — 인포그래픽의 핵심 요소
- 줄간격(line-height): 제목 1.15, 본문 1.6
- 영문 브랜드명: `letter-spacing: 0.08em` (넓은 트래킹)
- 한글 강조: `word-break: keep-all`

## 4. 섹션 구조 & 레이아웃

카드 전체 너비: **840px** (html2canvas 2× → 1680px PNG)

### 4.1 섹션 헤더 패턴

모든 섹션은 상단에 **4px 빨간 바(Bar)** + 섹션 제목을 붙인다.

```
┌────────────────────────────────────┐
│ ████ 섹션 타이틀          서브텍스트 │  ← 빨간 바 4px + H2
├────────────────────────────────────┤
│ 본문 콘텐츠                         │
└────────────────────────────────────┘
```

구현 패턴:
```tsx
<div style={{ borderTop: "4px solid var(--brand-accent)", paddingTop: 16 }}>
  <div style={{ fontSize: 11, fontWeight: 700, color: "var(--brand-accent)",
                letterSpacing: "0.1em", textTransform: "uppercase" }}>
    PERFORMANCE
  </div>
  <div style={{ fontSize: 22, fontWeight: 800, color: "#111" }}>
    성능 지표
  </div>
</div>
```

### 4.2 섹션 순서 (권장)

```
1. HERO          — 풀블리드 다크, 타이어 이미지 + 브랜드/모델명/태그라인
2. CLASSIFICATION — 차종 · 계절 · 등급 배지 (3열 그리드)
3. KEY SPECS     — 가로 스크롤 스펙 칩
4. FEATURE ICONS — 2×2 아이콘 그리드 (제동력·정숙성·연비·승차감)
5. POINT LAYOUT  — 타이어 이미지 중심, 좌우 POINT 카드 (최대 3×2)
6. PERFORMANCE   — 레이더 차트 or 막대 비교 차트
7. TECH          — 번호 있는 기술 설명 카드
8. RECOMMEND     — 추천 대상 카드
9. RENTAL PROCESS — 4단계 렌탈 서비스 프로세스 (스텝 인디케이터)
10. OPTIONS      — 사이즈별 옵션 테이블
11. FOOTER       — 다크 바, 브랜드 로고 + 면책
```

## 5. 컴포넌트 패턴 라이브러리

### 5.1 섹션 헤더 바 (SectionHeader)

```
[빨간 4px 상단 바]
ENGLISH LABEL    ← 10–11px, 700, accent color, uppercase
한국어 제목       ← 22px, 800, #111
```

### 5.2 분류 배지 (ClassificationBadge)

```
┌──────────────┐
│    🚗 아이콘  │
│  SUV / 승용   │
│  차량 구분    │
└──────────────┘
```
- 배경: `#f7f7f7`
- 선택 상태: 배경 `var(--brand-accent)`, 텍스트 `#fff`
- 크기: `148 × 96px`
- 아이콘 크기: 28px

### 5.3 스펙 칩 (SpecChip)

```
[ M+S  전천후 주행 ]
```
- 코드 영역: 배경 `#111`, 텍스트 `#fff`, 폰트 900, 12px
- 레이블 영역: 배경 `#f7f7f7`, 텍스트 `#444`, 폰트 500, 12px
- 높이: 32px, 둥글기: 4px

### 5.4 피처 아이콘 카드 (FeatureIconCard) — 2×2 그리드

```
┌─────────────┬─────────────┐
│  🛑          │  🔕         │
│  제동력      │  정숙성      │
├─────────────┼─────────────┤
│  ⛽          │  💺         │
│  연비 효율성  │  승차감      │
└─────────────┴─────────────┘
```
- 구분선: `1px solid #e8e8e8` (행/열)
- 각 셀 패딩: `24px`
- 아이콘 영역: 44px 원형 배경 `#f0f0f0`
- 레이블: 14px, 600

### 5.5 POINT 카드 레이아웃 (PointLayout)

레퍼런스 이미지 기반. 타이어 이미지 중앙 + 좌우에 포인트 텍스트.

```
┌──────────┬──────────────┬──────────┐
│ POINT 01 │              │ POINT 04 │
│ 제목      │  [타이어이미지]│ 제목      │
│ 설명...   │              │ 설명...   │
├──────────┤              ├──────────┤
│ POINT 02 │              │ POINT 05 │
│ 제목      │              │ 제목      │
│ 설명...   │              │ 설명...   │
└──────────┴──────────────┴──────────┘
```

- POINT 넘버: 8px, 700, `var(--brand-accent)`, `letter-spacing: 0.15em`
- 제목: 15px, 700, #111
- 설명: 12px, 400, #666
- 구분선: `1px solid #e8e8e8`
- 중앙 이미지: `240px × 280px`, `object-fit: contain`

### 5.6 성능 차트 (PerformanceBar)

레이더 차트 대신 **수평 바 차트**를 권장 (html2canvas 렌더링 안정성).

```
제동력   ██████████░░  4.5/5
정숙성   ████████████  5.0/5
핸들링   ████████░░░░  4.0/5
```

- 바 높이: 8px, 둥글기: 4px
- 주 제품 바 색상: `var(--brand-accent)`
- 비교 제품 바 색상: `#e0e0e0`
- 수치 레이블: 13px, 700, 우측 정렬

### 5.7 타이어 사이즈 다이어그램 (TireSizeViz)

```
     225 / 45 R 17
     ────────────
단면폭  편평비  림직경
```

- 각 수치: 36px, Black(900), `#111`
- 단위 레이블: 10px, 400, `#888`
- 슬래시·공백 구분: 24px, 300, `#ccc`

### 5.8 렌탈 프로세스 (RentalProcess) — 4단계 스텝 인디케이터

```
RENTAL PROCESS
렌탈 서비스 이용 안내

  [●]──────[○]──────[○]──────[○]
STEP 01   STEP 02   STEP 03   STEP 04
상담 신청  심사·승인  타이어 장착  렌탈 시작
설명...   설명...   설명...    설명...

[ 렌탈 혜택 안내 배너 — 초기 비용 0원·무상 펑크 수리 등 ]
```

- 현재 단계(01): 빨간 채움 원 + 빨간 텍스트 + 드롭 섀도
- 나머지 단계: 흰 원 + 회색 테두리
- 연결선: 빨간 → 투명 그라디언트, `position: absolute`
- 하단 혜택 배너: `accentSoft` 배경 + 좌측 4px 빨간 바
- 고정 데이터 (어드민 편집 불필요): 프로세스는 서비스 공통 플로우라 하드코딩

## 6. 어드민 데이터 → UI 매핑

어드민 패널에서 등록하는 필드가 카드의 어느 섹션에 표시되는지 정의.

| 어드민 필드 | 표시 위치 | 컴포넌트 |
|-------------|-----------|----------|
| 브랜드명 | HERO 상단 | `BRAND LABEL` 배지 |
| 모델명 | HERO 중앙 | Display 타입 (72px Black) |
| 태그라인 | HERO 하단 | H3, onDarkSoft |
| 차종 (carType) | CLASSIFICATION | 배지 (선택 강조) |
| 계절 (season) | CLASSIFICATION | 배지 (선택 강조) |
| 등급 (grade) | CLASSIFICATION | 배지 (선택 강조) |
| 퀵스펙 코드/레이블 | KEY SPECS | SpecChip |
| 피처 아이콘 레이블 | FEATURE ICONS | 2×2 아이콘 카드 |
| 기술 포인트 (checkpoints) | POINT LAYOUT | 좌우 배치 카드 |
| 성능 수치 (performance) | PERFORMANCE | 수평 바 차트 |
| 비교 수치 (comparison) | PERFORMANCE | 이중 바 차트 |
| 추천 대상 (recommendItems) | RECOMMEND | 이미지+텍스트 카드 |
| 옵션 (size/season) | OPTIONS | 테이블 행 |
| 히어로 이미지 | HERO | 풀블리드 배경 |
| 상품 이미지 | POINT LAYOUT 중앙 | 중앙 타이어 이미지 |

## 9. AI 슬롭 금지 패턴

다음 패턴은 AI가 자동으로 추가하는 "슬롭"이므로 **사용 금지**.

| 금지 패턴 | 이유 |
|-----------|------|
| 테이블 행 좌측 컬러 보더 | 과도한 장식, 밀도 저하 |
| 섹션마다 드롭 섀도 카드 | 중첩 그림자로 산만함 |
| 그라디언트 버튼 | 평평한(flat) 디자인 원칙 위반 |
| 아이콘 남발 (모든 레이블에 아이콘) | 정보 계층 훼손 |
| 인포박스에 emoji | 비전문적 인상 |
| 과도한 border-radius (> 12px) | 타이어 전문 브랜드 톤과 불일치 |
